## Formato de Documentación

Todos los manuales deben incluir:
- **Título descriptivo**
- **Tabla de contenidos**
- **Requisitos previos**
- **Instrucciones paso a paso**
- **Screenshots cuando sea necesario**
- **Troubleshooting común**
- **Referencias adicionales**

## Plantilla de Manual

```markdown
# [Título del Manual]

## Descripción
Breve descripción del propósito del manual.

## Requisitos Previos
- Requisito 1
- Requisito 2

## Instrucciones

### Paso 1: [Título]
Descripción e instrucciones

### Paso 2: [Título]
Descripción e instrucciones

## Screenshots
[Incluir capturas de pantalla relevantes]

## Troubleshooting

### Problema 1
**Síntoma**: Descripción
**Solución**: Pasos para resolver

## Referencias
- [Enlace 1]
- [Enlace 2]

## Última Actualización
Fecha: DD/MM/YYYY  
Por: [Nombre]
```

## Mantenimiento

Los manuales deben actualizarse cuando:
- Hay cambios en la UI/UX
- Se agregan nuevas funcionalidades
- Se modifican procesos
- Se recibe feedback de usuarios
